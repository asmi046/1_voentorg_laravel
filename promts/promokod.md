# Модель для промокода

Реализуй модель Promocod к ней создай миграцию сидер и контроллер.

## Поля необходимые для таблицы в базе данных

- Тип промокода - строковое обязательное поле
- Промокод - строковое обязательное поле
- Действуес с - дата nullable
- Действуес по - дата nullable
- Процент скидки - целочисленное по умолчанию 0
- Фиксированная сумма скидки - целочисленное по умолчанию 0
- Сумма с которой действует промокод - целочисленное по умолчанию 0

Поля помести в модель и миграцию. В миграции обязательно добавь комментарии.

## Создание сидера

Сделай сидер который заполни следующими данными:

- Тип промокода - Регулярный
- Промокод - OZON
- Действуес с - null
- Действуес по - null
- Процент скидки - 10
- Фиксированная сумма скидки - 0
- Сумма с которой действует промокод - 0

Сидер должен перезаписывать всю таблицу а не просто добавлять запись.

## Контроллер для проверки промокода

В контроллере реализуй метод верификации промомкода. Логика работы следующая:

В метод передаются следующие параметры:

- Промокод
- Сумма корзины

Далее поисходит каскад проверок:

- Проверяем наличие промокодав в базе (нужно реализовать регистронезависимую проверку) если нет возвращаем соответствующий код http ошибки
- Если промокод найден в базе проверяем:
    - В промокоде указанны диапоазон действия если не указан переходим к следующей проверке если указан проверяем входит ли текущая дата в этот диапазон если нет то ошибка если да то переход к следующей проверке
    - указанна ли сумма с которой действует промокод если нет переход к следующей проверке если да то проверяем переданная сумма корзины больше или нет, если больше переход к следующим действиям если меньше код ошибки

Далее проверяем если поле процент скидки отлично от нуля то вычисляем скидку в рублях от суммы корзины если нет то вычитаем из переданной суммы корзины поле Фиксированная сумма скидки.

Метод возвращает структуру в которо будет имя промокода и рассчитанная скидка.

# Изменения в корзине для применения промокода

Нужно внести изменения в компонент resources/js/components/cart/Cart.vue для применения промокода.

Логика следующая при вводе промокода к соответствующее поле и нажатия применить происходит запрос к коуту /promocod/verify. в него передается промокод и итоговая сумма корзины полученные данные запоминаются. Пользователю сообщается об ошибке или успешном примении промокода.

Если промокод применен в блоке <div class="itogo"> появется новая строчка с суммой скидки.

При изменении в корзине промкод пересчитываетс.
